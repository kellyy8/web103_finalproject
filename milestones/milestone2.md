## Reflection

### 1. What went well during this unit?

Before wireframing, we planned out the pages that we wanted to implement as well as where/how each CRUD operation should show up. We referenced our past projects to gain some insight on how the user flow should look like. By outlining our vision first, we were able to translate our ideas into a visual prototype smoothly. As for the entity relationship diagram, we followed a similar format by brainstorming all the data we would need and then organizing the information into an optimal number of tables. We communicated efficiently and effectively to better translate our ideas into tables and a diagram. We focused on ideation first, then creating the visuals.

### 2. What were some challenges your group faced in this unit?

Some challenges we faced include deciding what columns we should have in which table and how fine-grain we should divide our data into multiple tables. We thought about the latter in terms of optimization. We wanted the tables to feel like separate entities but not too fine-grained such that getting the information about any item or game would take a lot of joins to grab all the data that we need. We also wanted to ensure our project was feasible. The more columns we added, especially to describe the details of each item, the more filtering and management of data we would need to plan to account for.

We don't anticipate any major needs right now, but we plan to set up issues in a kanban board format to ensure that we work asynchronously smoothly.
