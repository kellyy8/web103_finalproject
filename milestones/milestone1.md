## Reflection

### 1. What went well during this unit?

During this unit, our group was able to quickly agree on a clear project idea: GameVault, a general inventory manager for items from any video game. We translated the course requirements into concrete features, which made it easier to write our README sections and align the feature list with the baseline and custom features we need to implement. We also collaborated smoothly on planning the user stories and making sure they matched real user flows, like managing items, loadouts, and filters, so the planning documents feel cohesive and realistic.

### 2. What were some challenges your group faced in this unit?

One challenge we faced was balancing our ambition for the app with the need to keep the scope manageable for the time we have. There are many interesting things we could do with a game inventory concept, so we had to be intentional about which features to prioritize for the final project. Another challenge was making sure our database and feature ideas would support the required relationships and API operations later; we spent extra time thinking through how games, items, and loadouts should connect so we do not paint ourselves into a corner in future units.

### 3. What additional support will you need in upcoming units as you continue to work on your final project?

In upcoming units, we may need additional support around designing and querying more complex PostgreSQL relationships, especially the many-to-many join table between loadouts and items. We would also benefit from guidance on best practices for structuring our Express routes and controllers so that our API stays organized as we add more features. Finally, help with debugging deployment issues on Render and confirming that our frontend and backend are correctly wired together in production would be useful as we move from planning into building and deploying the app.
